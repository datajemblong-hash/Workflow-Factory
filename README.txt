# 🏭 Tujuan Utama Workflow-Factory

## Menjadi Meta-Agent
Tujuannya adalah bertindak sebagai "Agen yang Membuat Agen", di mana ia berfungsi sebagai Manajer Proyek Otomatis untuk proyek Anda yang lain.

## Menyusun Fondasi Proyek
Pabrik ini bertujuan untuk menyusun konsep, fondasi, dan logika proyek secara otomatis, bukan untuk langsung menulis kode aplikasi (seperti C++ atau HTML).

## Efisiensi Waktu dan Konsistensi
Daripada Anda harus bersusah payah mengetik instruksi panjang setiap kali memulai proyek baru, pabrik ini memungkinkan Anda mendapatkan paket lengkap workflow siap pakai hanya dalam hitungan menit.

## Bersifat Universal
Pabrik ini dirancang agar 100% fleksibel dan bisa digunakan di berbagai platform, mulai dari mikrokontroler perangkat keras (IoT), pembuatan website, hingga aplikasi seluler.

# 🛠️ Fungsi Utama Workflow-Factory

## Melakukan Wawancara Adaptif
AI akan melakukan "Wawancara Dua Tahap" untuk menanyakan ide spesifik Anda dan platform utamanya (IoT, Web, Android, atau Desktop).

## Mendeteksi Kesenjangan Alat & Kebutuhan
AI akan menyesuaikan pertanyaannya; misalnya jika Anda memilih platform IoT, ia akan menanyakan ketersediaan komponen fisik. AI juga mendeteksi kebutuhan backend/API dan database secara otomatis.

## Mencetak Dokumen Standar & Panduan Baru
Menggunakan "tangan robotnya" (`cetak-dokumen-skill`), pabrik ini berfungsi mengeksekusi pembuatan folder baru dan mencetak dokumen fisik krusial:
- `README.md` (Utama)
- `1_PRD.md` (Daftar Fitur)
- `2_Arsitektur.md` (Desain & Mitigasi Risiko)
- `3_Daftar_Belanja.md` (Hanya jika ada komponen fisik belum dimiliki)
- `4_Troubleshooting.md` (Solusi 3 error umum platform)
- `5_Panduan_Deployment.md` (Panduan hosting & flash)
- `6_Skenario_Testing.md` (Test case dari fitur PRD)
- `7_Dokumentasi_API.md` (Kondisional — jika ada API)
- `8_Database_Schema.md` (Kondisional — jika ada Database)
- `9_Git_Workflow.md` (Branching & konvensi commit)
- `10_Security_Checklist.md` (Checklist audit keamanan platform)
- `11_Panduan_Setup.md` (Instalasi tools lokal)
- `status.md` (Sprint timeline harian)
- `agents.md` (Aturan AI developer proyek baru)

## Menyuntikkan Alat Otomatisasi (13 Skills)
Pabrik ini sekalian mencetak 11-13 folder skill operasional ke dalam proyek baru Anda agar siap digunakan untuk asisten AI berikutnya:
- `log-skill` & `git-skill` (Operasional)
- `review-kode-skill`, `deployment-skill`, `testing-skill`, `git-workflow-skill`, `security-audit-skill` (Kualitas & Panduan)
- `sprint-update-skill`, `backup-skill`, `env-setup-skill`, `readme-generator-skill` (Manajemen Proyek & Keselamatan)
- `error-solver-skill` & `prompt-helper-skill` (Vibe Coding Support)
- `dokumentasi-api-skill` & `database-schema-skill` (Kondisional)

## Menginisiasi Boilerplate Kode Dasar
Membuat folder dan file coding dasar secara otomatis (misalnya `main.cpp` dan `platformio.ini` untuk ESP, atau `index.html` dan `style.css` untuk Web) agar proyek 100% siap diketik.
